Resumen de comandos

En esta sección se presentan algunos de los comandos principales de Docker utilizados para construir, ejecutar y administrar contenedores durante el desarrollo del laboratorio.

Comandos básicos y su función

1. docker run hello-world

docker run hello-world

Permite comprobar que Docker está instalado y funcionando correctamente.

2. docker build -t mi-web-html .

docker build -t mi-web-html .

Construye una imagen Docker a partir del Dockerfile ubicado en la carpeta del proyecto.

3. docker run -d -p 8080:80 --name mi-web-container mi-web-html

docker run -d -p 8080:80 --name mi-web-container mi-web-html

Ejecuta el contenedor en segundo plano y publica el sitio web en localhost:8080.

4. docker ps

docker ps

Muestra los contenedores que se encuentran actualmente en ejecución.

5. docker ps -a

docker ps -a

Muestra todos los contenedores, incluidos los que están detenidos.

6. docker stop mi-web-container

docker stop mi-web-container

Detiene un contenedor que se encuentra en funcionamiento.

7. docker rm mi-web-container

docker rm mi-web-container

Elimina un contenedor detenido para poder recrearlo nuevamente.

8. docker images

docker images

Lista las imágenes Docker disponibles en el equipo.

9. docker logs mi-web-container

docker logs mi-web-container

Permite revisar los registros o mensajes generados por el contenedor.

10. docker pull nginx:alpine

docker pull nginx:alpine

Descarga una imagen desde Docker Hub para utilizarla localmente.

Importancia de estos comandos

Estos comandos permiten gestionar el ciclo completo de trabajo con Docker: verificar su instalación, construir imágenes, levantar contenedores, revisar su estado y eliminarlos cuando sea necesario. Su uso facilita el despliegue de sitios web de forma ordenada y controlada.
